- **Чат:** Панель чата справа (mock, готов к подключению LiveKit Data Channel)
- **Тема:** Переключение тёмной/светлой, сохранение в localStorage

### v0.5.1 — MAX OAuth flow
- Убран хардкодный неверный URL (punycode баг с кириллическим доменом)
- Реализован полный flow: `max-auth-init` → deep link в браузер → поллинг → создание комнаты → LiveKit токен

- Переключение на punycode домен (`xn--e1ajhcbd3acj.xn--p1ai`)
- Добавлен `tauri-plugin-http` для обхода ограничений WKWebView
- Добавлен `com.apple.security.network.client` entitlement

### v0.6.0 — Авторизация, кроп, захват, ссылка, запись
- **Авторизация через Rust/reqwest** — обходит WKWebView ограничения macOS

## Ответы на вопросы пользователя

### 1. Связь приложения и веб-платформы
Да, приложение и вебинарная платформа связаны через LiveKit:
- Ведущий подключается через приложение → LiveKit SFU
- Зрители подключаются через веб-интерфейс → тот же LiveKit SFU
- Ссылка для зрителей ведёт на `контентум.рф/vebinar/room/{id}`

### 2. Снижение нагрузки
Да, нативное приложение снижает нагрузку:
- **На сервер:** WebRTC эффективнее, чем браузерный WebRTC (меньше overhead)
- **На компьютер ведущего:** Нет нагрузки от браузера (табы, расширения, JS-движок). Tauri использует ~50-80MB RAM vs Chrome ~500MB+
- **Запись на сервере:** LiveKit Egress записывает на сервере — 0% нагрузки на ПК ведущего
- **В будущем (Фаза 3):** Аппаратное кодирование (VideoToolbox на Mac, NVENC на Nvidia) снизит нагрузку на CPU на 80-90%
